An ECU (Electronic Control Unit) Build is a specialized software configuration for automotive SoCs, designed to optimize system resources and power consumption for real-world vehicle applications. In the context of the J722S device running QNX+RTOS, the ECU Build enables selective activation of only the necessary cores and peripherals required for a specific demo or use-case, while disabling unused components (Core/LPSC) to minimize power usage and thermal footprint.

  • Power Optimization: By disabling unused cores or peripherals and optimizing core frequency, the system consumes less power, which is critical for automotive and edge applications where energy efficiency is paramount.
  • Thermal Management: Lower power consumption leads to reduced heat generation, improving system reliability and longevity.
  • Automatic Demo Startup: The ECU Build configuration ensures that the demo starts automatically on boot. This initializing the necessary components and launching the demo application without requiring manual intervention.
  1. Set ECU_BUILD Flag:
    Edit sdk_builder/makerules/makefile_ecu.mak.
    ECU_BUILD?=no
    # Change to
    ECU_BUILD?=fc
    'fc' stands for Front Camera. This ensures only required components for the front camera demo are enabled.
  2. **(Optional) Update Clock and Device Configuration:**
    • Use the syscfg tool to fine-tune clock and device settings for SBL (Secondary Boot Loader).
      cd mcu_plus_sdk
      make -C examples/drivers/boot/sbl_sd_hlos_fc/j722s-evm/wkup-r5fss0-0_nortos/ti-arm-clang/ syscfg-gui
      # or
      make -C examples/drivers/boot/sbl_ospi_hlos_fc/j722s-evm/wkup-r5fss0-0_nortos/ti-arm-clang/ syscfg-gui
      For more info refer to the DM Power Configuration Tool User Guide.
  1. Build the SDK:
    Navigate to sdk_builder and build:
    cd sdk_builder
    make sdk -j
  2. Prepare SD Card for QNX Boot:
    Follow instructions at Prepare SD card for boot (one time only).
  3. Copy QNX Filesystem and Binaries to SD Card:
    make qnx_fs_create_sd
    make qnx_fs_install_sd # When ECU Build flag is enabled, this will copy auto run scripts and config files for front camera demo.
  4. Build and Install SBL Images: Refer to SBL Boot Support with QNX+RTOS mode, ensuring ECU_BUILD is set to 'fc'.
  5. Prepare Hardware:
    Connect the IMX728 camera to the J7 EVM board via the CSI interface.
  6. Run The Demo:
    Boot the board with the prepared SD card. The front camera demo should start automatically, utilizing only the necessary cores and peripherals.
